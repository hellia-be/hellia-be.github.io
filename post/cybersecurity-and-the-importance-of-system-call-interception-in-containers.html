Cybersecurity and the Importance of System Call Interception in Containers

March 28, 2023 Kevin Andriessens

In today's digital landscape, the security of applications and data is more important than ever. With the increasing use of containers for application deployment, it is critical to ensure that these containers are secure from cyber threats.

One approach to increase the security of containers is through system call interception. System call interception involves intercepting system calls made by an application and modifying them before they reach the kernel. By doing so, we can prevent malicious system calls from being executed, thereby reducing the risk of a security breach.

The initial targets for interception are key system calls that pose the greatest security risks. For example, the mknod and mknodat system calls, which are used when running docker inside an LXD container, can be intercepted to prevent unauthorized access to the host system. Similarly, the setxattr system call can be intercepted to prevent malicious actors from modifying file permissions or ownership, which could lead to data theft or unauthorized access.

Another important system call is mount, which is commonly used to mount disks. However, this system call can be unsafe, and therefore, a safer alternative, fuse2fs, is being used to ensure the security of the host system.

The sched_setscheduler system call is also critical from a cybersecurity perspective. This system call is needed for the android container, but it can also make the container unkillable if not handled correctly. By intercepting this system call, we can prevent attackers from creating unkillable containers, which could lead to a persistent breach.

Finally, the sysinfo system call is also being targeted due to the fact that some systems, such as Alpine, do not follow standards, which could result in security risks. By intercepting this system call, we can prevent malicious actors from exploiting vulnerabilities in non-standard systems.

In conclusion, system call interception is a crucial aspect of securing containers in today's cybersecurity landscape. By targeting key system calls that pose the greatest security risks, we can significantly reduce the risk of a security breach and ensure the safe deployment of applications in containers.
